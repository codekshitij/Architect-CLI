Architect-CLI v0.2 Change Log
Date: 2026-03-06

Overview
- Refactored core execution flow into a clearer pipeline.
- Improved dependency resolution correctness and reduced false matches.
- Added safer/more flexible CLI options.
- Hardened visualizer node identity and Mermaid security mode.
- Updated README usage examples to match new CLI behavior.

Files Changed
1) main.py
2) architect/visualizer.py
3) README.md

Detailed Changes

1) main.py
- Replaced monolithic main() logic with staged functions:
  - discover_files(root_path)
  - build_indexes(paths)
  - extract_dependency_candidates(dep_text)
  - resolve_edges(raw_deps, by_filename, by_stem)
  - apply_focus(edges, focus)
  - label_edges(edges, file_cache, brain, no_llm)
  - build_parser()

- Added path safety and argument validation:
  - Validates --path exists.
  - Validates --path is a directory.
  - Validates --max-edges >= 0.

- Added new CLI options:
  - --output (custom HTML output path)
  - --max-edges (limit number of edges analyzed)
  - --no-llm (skip model labeling and use generic label)
  - --open / --no-open (control browser launch)

- Improved file discovery behavior:
  - Uses directory pruning for ignored folders:
    - .git
    - node_modules
    - venv
    - __pycache__

- Fixed filename collision bug:
  - Previous approach keyed files by basename only (could overwrite files with same name in different folders).
  - New approach indexes by full path and keeps lookup maps by filename/stem.

- Improved dependency resolution strategy:
  - Extracts candidate tokens from import/include strings.
  - Normalizes candidates using basename/stem/path segments.
  - Resolves against by_filename and by_stem maps.
  - Deduplicates edges via set and returns stable sorted output.

- Output behavior:
  - Respects --output path.
  - Creates output directory if needed.
  - Opens browser only when open_browser is enabled.

2) architect/visualizer.py
- Added hashlib import.
- Changed Mermaid security setting:
  - securityLevel: 'loose' -> 'strict'

- Fixed node ID collision risk:
  - Previous ID generation used sanitized basename only.
  - New ID generation uses sanitized basename + short MD5(path).
  - Prevents collisions when files share same basename in different directories.

3) README.md
- Replaced old commented usage snippet with real command examples.
- Added examples for:
  - Normal run with --output
  - No-LLM run with --no-open
  - Focused run with --focus and --max-edges

Date: 2026-03-06

Files Changed

Detailed Changes

1) main.py
- Added persistent JSON cache support:
  - DEFAULT_CACHE_FILE = .architect_cache.json
  - load_cache(cache_file)
  - save_cache(cache_file, cache_data)
  - make_cache_key(model, source_path, source_code, target_path, target_code)
  - Cache key includes model + file paths + code hashes for invalidation when code changes.

  - scan_files(scanner, all_paths, workers)
  - Uses ThreadPoolExecutor + as_completed when workers > 1.

- Added parallel labeling:
  - label_edges(..., workers, use_cache, cache_data)
  - Reuses cached labels when available.
  - Labels unresolved edges concurrently when workers > 1.

- Added worker resolution and new CLI flags:
  - --workers (0 = auto, otherwise exact thread count)
  - --cache-file (path to JSON cache)
  - --no-cache (disable cache read/write)

- Added argument validation:
  - --workers must be >= 0

2) architect/brain.py
- Replaced bare exception handling with explicit failure paths.
- Added configurable transport controls:
  - timeout
  - max_retries
  - retry_backoff_seconds
- Added response.raise_for_status() and JSON parse handling.
- Added linear backoff between retries.

3) README.md
- Added examples for:
  - parallel workers + cache file
  - disabling cache for debugging
